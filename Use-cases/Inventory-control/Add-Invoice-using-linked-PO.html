Add-Invoice-using-linked-PO

Basic Path

  1. The software checks that the invoice's PO is linked to a prior order.

  2. From the PO number, the software retrieves the priorly ordered products, that is, the product's identifying information ordered quantity and units

  3. For each product on the order

    1. The software prommpts the Inventory Manager for the shipped quantity.

    2. The Inventory Manager enters the shipped quantity. The default is 0.

  4. The software allows the Inventory manager to add another product that may have been delivered but was not on the order. This adding of products is done similary as in adding an invoice not linked to a PO.

  5. The software checks that the added product is not already on the invoice. If it is, the priorly entered product is shown to the Inventory Manager and the additional add is aborted. If the addition is done using a barcode scanner the priorly added product is highlighted and the additional quantity shipped is added to the prior quantity.

  6. Differences between the shipped quantity and the ordered quantity are highlighted.
