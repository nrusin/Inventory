Add PO

Basic Path

  1. The use case begins when the Inventory Manager selects "Add PO".

  2. The software retrieves the possible vendors to use.

  3. The software checks that there is at least one vendor.

  4. If there is only one possible vendor, the software goes ahead and uses that vendor.

  5. The software prompts for the PO number.

  6. The software promtps for the vendor.

  7. While the Inventory Manager is not done adding products

    1. The software prompts for the product.

    2. The Inventory Manager enters the product, either by entering the product's SKU, UPC, last 6 digits of UPC, or description.

    3. The software checks that the product is not already on the purchase order.

    4. The software retrieves the default unit type for the product selected.

    5. The software prompts the Inventory Manager for the unit to use, using the default.

    6. The software prompts the Inventory Manager for the quantity ordered.

    7. The Inventory Manager enters the quantity and unit type of the product ordered.

  8. After the Inventory Manager is done adding products, the software prompts whether to add the PO.

  9. The Inventory Manager chooose to add the PO.

  10. The software adds the PO.

  11. The software updates any listing of POs.
