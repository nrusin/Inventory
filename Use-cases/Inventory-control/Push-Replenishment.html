Push Replenishment

Basic Path

  1. The use case begins when the Inventory Manager selects "View Replenishments".

  2. The Inventory Manager selects "Add Replenishement".

  3. The software prompts the Inventory Manager for the replenishment's filename.

  4. The Inventory Manager enters the replenishment's filename.

  5. The software reads the replenishment.

  6. The software checks that the replenishment is consecutive with prior replenishments.

  7. For each product within the replenishment

    1. The software checks that the product's upc is valid.

    2. The software looks up the product by upc. If the product within the replenishment is not found the software creates a new product with upc.

    3. The software enters the product and the quantity sold into the replenishment.

  8. The software shows the Inventory Manager the list of products that were added.
